# Queue Management System – QA Testing

## Objective
To validate data integrity and backend behavior of a queue management system by testing UI interactions, data storage, and system responses.

---

## System Overview
- Frontend: Web-based client sign-in & staff dashboard
- Backend: Google Apps Script
- Database: Google Sheets

---

## Testing Focus
- Data validation (database consistency)
- API behavior (Apps Script execution)
- End-to-end workflow validation

---

## Key Features Tested
- Client sign-in
- Queue creation
- Status updates (Waiting → In Office → Ready)
- Queue reset functionality
